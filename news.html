Proprietà "catch-action" sulle azioni
19.10.2023 | Janox news

In release di sviluppo 2.9.02 è stata introdotta la possibilità di indicare una "catch-action" sulle azioni.

Se viene specificata una "catch-action", allora a qualunque verificarsi di errore o exception nell'azione (comprese execute-action, call-program, ecc...), il controllo verrà passato all'azione di catch, che potrà gestirli a piacimento.

Questa modifica introduce un modello completamente nuovo di gestione di errori/eccezioni in Janox.

 

Integrazione di Git
20.09.2023 | Janox news

In release di sviluppo 2.9.02 è stata introdotta l'integrazione con il sistema di versioning Git .

Quando l'integrazione è attiva il JX Open Builder, al check-in di ogni elemento, richiederà di fornire un messaggio ed eseguirà la relativa commit su Git.

Per attivare l'integrazione basterà abilitare il flag "Git" in "Preferences" del JX Open Builder e attivare Git nella root dell'applicazione Janox.

 

Nuovo controllo File-Upload
03.08.2023 | Janox news

In release di sviluppo 2.9.01 è stato introdotto il nuovo modello per i controlli di tipo File-Upload, che prevede l'utilizzo delle proprietà "multiple", "submit-on-change" e "on-change-action".

La proprietà "multiple" consente di selezionare più file per l'upload.

Se viene abilitata la proprietà "submit-on-change" il controllo viene visualizzato come Upload-Button e servirà un singolo click per selezionare ed inviare i file.

La proprietà "on-change-action" consente di indicare un'azione per processare i file caricati al singolo click.

 

SAML2 Login
03.07.2023 | Janox news

In release di sviluppo 2.9.01 è stato introdotto il login su protocollo SAML2 .

Il nuovo protocollo di login prevede l'utilizzo della validazione SAML2 da terze parti, per accedere alle applicazioni Janox.

La funzionalità si appoggia sull'installazione e la gestione della libreria SimpleSAMLPHP.

Prepared statements
29.08.2022 | Janox news

La versione in sviluppo 2.8.00 introduce la possibilità di ottimizzare le viste per le letture multiple, attraverso l'utilizzo di prepared statements.

La gestione dei parametri è totalmente automatizzata e il programmatore deve solo attivare il relativo flag sulle proprietà della vista.

Questa funzionalità permette di ridurre i tempi di query di oltre il 50%, quando vengono eseguite molte letture sulla vista, ad esempio all'interno di un loop.

Indici non univoci nelle viste
27.12.2020 | Janox news

La versione in sviluppo 2.6.03 introduce la possibilità di utilizzare anche gli indici non univoci, oltre a quelli univoci, come chiave di ordinamento nelle viste.

Adesso sarà possibile, quindi, definire sulle tabelle molti indici come non univoci (senza i segmenti di PK aggiunti), guadagnando sia sulla dimensione degli indici creati, che sui tempi di scrittura delle tabelle stesse.

In caso di indici non univoci la runtime garantisce comunque le piene funzionalità della vista.

JXSQL Custom WHERE clause
22.06.2020 | Janox news

La versione in sviluppo 2.6.02 introduce un nuovo potente strumento per la definizione delle viste: una clausola WHERE custom, codificabile in JXSQL.

Questo consentirà di definire espressioni SQL, anche complesse, per definire criteri di range personalizzati, anche oltre le possibilità standard delle range sulle singole SELECT (OR, EXISTS, IN, ...), senza dover ridefinire l'intero sorgente per la vista.

 

JXSQL Fields
03.06.2020 | Janox news

La versione in sviluppo 2.6.02 introduce un nuovo potente strumento: i campi JXSQL..

Adesso sarà possibile definire un sorgente SQL specifico per un singolo campo, in portabilità, tramite JXSQL..

Questo consentirà di definire espressioni SQL, anche complesse, per definire uno o più campi, senza dover ridefinire tutto il sorgente del dataset come sub-select (table expression).

Database As a Service
02.03.2020 | Janox news

La modalità Database As a Service è un nuovo potente strumento di connettività Janox.

Adesso è possibile definire un'entrata ad una applicazione Janox, in modalità RPC, che operi come un service per l'esposizione di un database.

Da altre applicazioni Janox sarà così possibile definire un database remoto (tramite il nuovo gateway JXSDB) ed utilizzarlo come un normale database locale.

 

Recordset snapshot
09.12.2019 | Janox news

Lo snapshot di recordset permette di spostare l'intero dataset intercettato da una vista su database SQLite session.

Il database SQLite session lavora "in memory" per la singola esecuzione di server, ma assicura la persistenza necessaria all'esecuzione di successive interazioni client.

Alla fine di una sessione di snapshot sarà possibile, programmaticamente, sincronizzare i dati modificati con il database originale o scartare l'intero set di modifiche, in un'unica transazione, senza ricorrere a tabelle temporanee.
